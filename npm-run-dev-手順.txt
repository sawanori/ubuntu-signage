==================================================================
 npm run dev が起動しないときの手順書
 （WSL2 / fish / tmux 環境）
==================================================================

■ 何が起きていた？
------------------------------------------------------------------
コードのバグではありません。
Electron（GUIアプリ）を表示する画面の出力先（DISPLAY）が
今のターミナルに設定されていなかったため、起動直後に落ちていました。

  エラー: Missing X server or $DISPLAY
          The platform failed to initialize. Exiting.

設定ファイル（config.fish）には正しい設定が入っていますが、
あなたが使っている tmux（ターミナル）が6月12日から起動しっぱなしで、
設定追加より前だったため、その設定を読み込めていませんでした。


==================================================================
 ★ いますぐ動かす（いちばん簡単）
==================================================================
今のターミナルにそのまま、次の1行を貼り付けて Enter：

    DISPLAY=:0 npm run dev

→ これでアプリが起動します。これだけで OK。


==================================================================
 ★ 毎回 DISPLAY=:0 を付けたくない場合
==================================================================
（A案）新しいタブ（ペイン）を開いてから実行する
------------------------------------------------------------------
  1) キーボードで  Ctrl + b  を押す → 続けて  c  を押す
     （tmux の新しいウィンドウが開きます）
  2) プロジェクトに移動：
        cd /home/noritakasawada/project/ubuntuapp
  3) そのまま実行：
        npm run dev
     → 新しいウィンドウは設定済みなので DISPLAY を付けなくてOK


（B案）今のターミナルを1回だけ直す
------------------------------------------------------------------
今のターミナルに、次の3行を順番に貼り付けて Enter：

    set -gx DISPLAY :0
    set -gx WAYLAND_DISPLAY wayland-0
    set -gx XDG_RUNTIME_DIR /mnt/wslg/runtime-dir

そのあと：

    npm run dev


==================================================================
 ★ 今後ずっと自動で直しておきたい場合
==================================================================
パソコン（WSL）を再起動するか、tmux を完全に再起動すれば、
設定ファイルが最初から読み込まれて、以後は何もしなくてOKになります。

  ・WSL再起動の例（Windowsのコマンドプロンプトで）:
        wsl --shutdown
    その後ターミナルを開き直す

  ※ tmux kill-server でも直りますが、開いている画面が
    すべて閉じるので注意。急がないなら WSL再起動が安全。


==================================================================
 ★ 起動後に出る赤いエラーについて（気にしなくてOK）
==================================================================
アプリ起動後、以下が繰り返し出ますが【正常】です：

  ・localhost:8080 ERR_CONNECTION_REFUSED
      → オーバーレイを重ねる「土台サイト」がまだ起動して
        いないだけ。アプリは落ちずに待機し続けます。
        動画オーバーレイ機能の確認は土台サイト無しでも可能。

  ・GPU process ... errors during initialization
      → WSLにグラフィック機能が無いための表示で、無害です。


==================================================================
 まとめ（迷ったらこれ）
==================================================================
       DISPLAY=:0 npm run dev

これを実行すれば動きます。
